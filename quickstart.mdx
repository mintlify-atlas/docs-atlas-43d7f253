This guide will help you set up and run the WhatsApp Message Scheduler service from scratch. By the end, you'll have a fully functional API connected to your WhatsApp account.

## Prerequisites

Before you begin, make sure you have:

- **Bun** installed (v1.0 or higher) - [Install Bun](https://bun.sh/)
- **A Redis instance** - You can get a free Redis database from [Upstash](https://upstash.com/)
- **A WhatsApp account** on your mobile device for QR code scanning
- **Git** for cloning the repository

<Warning>
This service requires access to WhatsApp Web. Make sure your WhatsApp account is active and you have access to your phone for QR code scanning during initial setup.
</Warning>

## Installation

<Steps>
  <Step title="Clone the repository">
    Clone the project to your local machine:
    
    ```bash
    git clone https://github.com/anupom69/bun-hono-backend-optimized.git
    cd bun-hono-backend-optimized
    ```
  </Step>

  <Step title="Install dependencies">
    Use Bun to install all required packages:
    
    ```bash
    bun install
    ```
    
    This will install:
    - Hono (web framework)
    - Prisma (database ORM)
    - BullMQ (job queue)
    - whatsapp-web.js (WhatsApp client)
    - Other necessary dependencies
    
    <Note>
    The `postinstall` script will automatically run `prisma generate` to generate the Prisma client.
    </Note>
  </Step>

  <Step title="Set up Redis">
    Get your Redis connection URL from Upstash:
    
    1. Sign up at [Upstash](https://upstash.com/)
    2. Create a new Redis database
    3. Copy the REST URL from the dashboard
    
    You'll need this URL for the next step.
  </Step>

  <Step title="Configure environment variables">
    Create an `.env` file in the root directory:
    
    ```bash
    touch .env
    ```
    
    Add the following configuration:
    
    ```env
    PORT=3000
    DATABASE_URL="file:./dev.db"
    REDIS_REST_URL="your_redis_url_here"
    USERNAME=user
    PASSWORD=pass
    ```
    
    <CodeGroup>
    ```env Production
    PORT=3000
    DATABASE_URL="file:./prod.db"
    REDIS_REST_URL="https://your-redis.upstash.io"
    USERNAME=admin
    PASSWORD=your_secure_password
    ```
    
    ```env Development
    PORT=3000
    DATABASE_URL="file:./dev.db"
    REDIS_REST_URL="http://localhost:6379"
    USERNAME=user
    PASSWORD=pass
    ```
    </CodeGroup>
    
    <Warning>
    **Security Notice**: Change the default `USERNAME` and `PASSWORD` values before deploying to production. These credentials protect your API endpoints with basic authentication.
    </Warning>
  </Step>

  <Step title="Initialize the database">
    Run Prisma migrations to set up your SQLite database:
    
    ```bash
    bunx prisma migrate dev --name init
    ```
    
    This creates the database schema with three tables:
    - `AdminProfile` - Stores admin information and last check-in time
    - `People` - Manages contacts with names and phone numbers
    - `Message` - Stores scheduled messages
  </Step>
</Steps>

## Running the Server

<Steps>
  <Step title="Start the server">
    Launch the application with:
    
    ```bash
    bun run start
    ```
    
    You'll see output similar to:
    
    ```
    [QR RECEIVED]
    ```
    
    Followed by a QR code displayed in your terminal.
  </Step>

  <Step title="Scan the QR code">
    1. Open WhatsApp on your mobile device
    2. Go to **Settings** → **Linked Devices**
    3. Tap **Link a Device**
    4. Scan the QR code displayed in your terminal
    
    <Note>
    The QR code expires after a short time. If it expires, restart the server to generate a new one.
    </Note>
  </Step>

  <Step title="Verify connection">
    Once successfully connected, you'll see:
    
    ```
    [CLIENT]: WhatsApp client is ready.
    [WORKER]: Ready!
    [SERVER]: Running on http://localhost:3000
    ```
    
    You should also receive a WhatsApp message on your phone:
    
    > Backend Server Started successfully! 🚀
    
    <Note>
    The confirmation message is sent to the phone number configured in `src/server.ts:34`. You may want to update this to your own number or remove this line.
    </Note>
  </Step>
</Steps>

## Testing Your Installation

Let's verify everything is working by testing the API endpoints.

<Steps>
  <Step title="Check server health">
    Test the base endpoint:
    
    ```bash
    curl http://localhost:3000/
    ```
    
    Expected response:
    
    ```json
    {
      "message": "Backend Server is running !!!"
    }
    ```
  </Step>

  <Step title="Create a contact">
    Add a contact to your database (requires authentication):
    
    ```bash
    curl -X POST http://localhost:3000/people/create-one \
      -u user:pass \
      -H "Content-Type: application/json" \
      -d '{
        "name": "John Doe",
        "phone": "1234567890123"
      }'
    ```
    
    <Warning>
    Phone numbers must be exactly 13 characters long, formatted as country code + number (e.g., `8801234567890` for Bangladesh).
    </Warning>
    
    Expected response:
    
    ```json
    {
      "message": "Person created successfully"
    }
    ```
  </Step>

  <Step title="Schedule a message">
    Create a scheduled message:
    
    ```bash
    curl -X POST http://localhost:3000/messages/create-one \
      -u user:pass \
      -H "Content-Type: application/json" \
      -d '{
        "content": "Hello from WhatsApp Message Scheduler!",
        "sendToPhone": "1234567890123",
        "sendAfter": 1
      }'
    ```
    
    The `sendAfter` field represents days from the last admin check-in. Messages scheduled for 1 day will be sent 24 hours after the reference timestamp.
    
    Expected response:
    
    ```json
    {
      "message": "Message created successfully"
    }
    ```
  </Step>

  <Step title="Send an instant message">
    Test immediate message delivery:
    
    ```bash
    curl -X POST http://localhost:3000/messages/send-instant \
      -u user:pass \
      -H "Content-Type: application/json" \
      -d '{
        "content": "Test message",
        "sendToPhone": "1234567890123"
      }'
    ```
    
    The message should arrive immediately on WhatsApp.
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="QR code not appearing">
    Make sure:
    - Your terminal supports UTF-8 encoding
    - No firewall is blocking WhatsApp Web connections
    - You have a stable internet connection
    
    Try restarting the server if the QR code doesn't appear within 10 seconds.
  </Accordion>
  
  <Accordion title="Messages not being sent">
    Check that:
    - The WhatsApp client shows as `[CLIENT]: WhatsApp client is ready.`
    - Redis connection is working (check the `REDIS_REST_URL` in your `.env`)
    - The phone number format is correct (13 characters)
    - The contact exists in your `People` table before scheduling messages
  </Accordion>
  
  <Accordion title="Authentication failing">
    Verify that:
    - You're using the correct `USERNAME` and `PASSWORD` from your `.env` file
    - Basic auth is properly formatted: `username:password`
    - The credentials are base64 encoded in the Authorization header
    
    Example with correct formatting:
    ```bash
    curl -u user:pass http://localhost:3000/people/all
    ```
  </Accordion>
  
  <Accordion title="Database errors">
    If you encounter Prisma errors:
    
    ```bash
    # Regenerate Prisma client
    bunx prisma generate
    
    # Reset database (warning: deletes all data)
    bunx prisma migrate reset
    
    # View your data
    bunx prisma studio
    ```
  </Accordion>
</AccordionGroup>
