## Base URL

All API requests should be made to:

```
http://your-server-url/
```

## Authentication

The API uses **HTTP Basic Authentication** for protected routes. You need to provide credentials for all `/messages` and `/people` endpoints.

Credentials are configured via environment variables:
- Username: `USERNAME` (defaults to "user")
- Password: `PASSWORD` (defaults to "pass")

### Example Authenticated Request

```bash
curl -X GET http://your-server-url/messages/all \
  -u username:password \
  -H "Content-Type: application/json"
```

Or using the Authorization header:

```bash
curl -X GET http://your-server-url/messages/all \
  -H "Authorization: Basic $(echo -n 'username:password' | base64)" \
  -H "Content-Type: application/json"
```

#### Endpoints Summary

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/messages/all` | List all messages with recipient details |
| POST | `/messages/create-one` | Create a new scheduled message |
| GET | `/messages/read-one/:id` | Get a specific message by ID |
| POST | `/messages/update-one/:id` | Update a scheduled message |
| POST | `/messages/delete-one/:id` | Delete a scheduled message |
| GET | `/messages/reschedule` | Reschedule all pending messages |
| POST | `/messages/send-instant` | Send a message immediately |

## Response Format

All endpoints return JSON responses with appropriate HTTP status codes:

- `200` - Success
- `400` - Bad Request (missing or invalid parameters)
- `404` - Not Found
- `500` - Internal Server Error

### Success Response Example

```json
{
  "message": "Message created successfully"
}
```

### Error Response Example

```json
{
  "message": "Missing required fields"
}
```

## Rate Limiting

Currently, there are no rate limits enforced on the API. However, it's recommended to implement rate limiting in production environments.

## CORS

Configure CORS settings according to your deployment requirements. The default Hono setup does not include CORS middleware.
